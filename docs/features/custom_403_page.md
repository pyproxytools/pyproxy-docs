A customization system for the 403 page for denied requests is available.
By default, the denial page is defined in the `assets/403.html` file.
You can edit this page directly or use your own page using a specific configuration.

## Change the file path
You can change the path to the 403 denial page. Here is the associated documentation, depending on your configuration.

* [From an .ini file](../configuration/from_ini_file.md)
* [With command argument](../configuration/command_argument.md)
* [Environment variables](../configuration/env_var.md)
